# GraphX.AI — GNN AutoML Platform for IC Layout Analysis

> Upload an Excel workbook → AutoML picks the best GNN → inspect, evaluate, predict.

GraphX.AI is a self-contained **Graph Neural Network (GNN) AutoML** platform aimed at
EDA / IC layout workflows. A guided 6-step pipeline (Create → Upload → Analyze →
Train → Evaluate → Predict) runs Optuna hyper-parameter search across a curated
model pool and surfaces the best configuration with full metric reports — no
ML expertise required.

---

## Tech stack

| Layer | Technology |
|-------|------------|
| Frontend | Next.js 16 (App Router), React 19, Ant Design 5, Recharts, Framer Motion |
| Backend  | FastAPI + Pydantic v2 |
| ML       | PyTorch, PyTorch Geometric, PyTorch Lightning |
| AutoML   | Optuna |
| Storage  | In-memory (no DB), pickled checkpoints under `backend/storage/` |
| Auth     | NextAuth + Keycloak OIDC (optional) |

The whole stack runs on a single laptop. CUDA is auto-detected; CPU works as
fallback for the demos.

## Quick start

```bash
# Backend (Python 3.11+, uv recommended)
cd backend
uv sync                        # or: pip install -r requirements.txt
.venv/Scripts/python -m uvicorn app.main:app --reload --port 8000

# Frontend (Node 20+)
cd frontend
npm install
npm run dev                    # http://localhost:3000
```

Smoke-test in 5 minutes:

1. Open `http://localhost:3000` → **New Project**.
2. On the Upload page, hit **Load** on `Multi-Graph Heterogeneous`.
3. Inspect graph counts / feature schema on the Explore page → **Confirm & Proceed to Training**.
4. Pick model families (defaults are sensible) → **Start Training**.
5. Evaluate page renders metrics, residual plot, history, and the HPO leaderboard.
6. Predict page reuses the trained model against new (or replayed) input.

## Excel data format (since 2026-04-24)

A single `.xlsx` workbook with four sheets:

| Sheet | Required columns | Notes |
|-------|------------------|-------|
| `Parameter` | `XY`, `Level`, `Type`, `Parameter`, optional `Weight` | Declares the schema. `XY` is `X` (feature) or `Y` (label). `Level` is `Node` / `Edge` / `Graph`. |
| `Node` | `_graph` (multi-graph) + node columns declared in Parameter | Add a `Type` column for heterogeneous graphs. |
| `Edge` | `_graph`, `src`, `dst` + optional edge columns | Add a `Type` column for heterogeneous edges (`cell_pin`, `pin_net`, …). |
| `Graph` | `_graph` + graph-level columns | Targets for `graph_classification` / `graph_regression`. |

A feature can be declared once per `Type` and appear under multiple types — the
backend treats it as the same logical feature with type-specific values
(useful for cross-type quantities like `area_um2` on both `cell` and `pin`).

Demo workbooks live in `backend/demo_data/` and are regenerated by
`backend/scripts/generate_excel_demos.py`.

## Supported task types

| Task | Backbones (homo) | Backbones (hetero) |
|------|------------------|--------------------|
| `node_classification` | GCN, GAT, GraphSAGE, GIN, MLP | GAT, GraphSAGE, MLP |
| `node_regression`     | GCN, GAT, GraphSAGE, GIN, MLP | GAT, GraphSAGE, MLP |
| `graph_classification`| GCN, GAT, GraphSAGE, GIN, MLP | GAT, GraphSAGE, MLP |
| `graph_regression`    | GCN, GAT, GraphSAGE, GIN, MLP | GAT, GraphSAGE, MLP |

GCN and GIN are excluded for heterogeneous datasets — neither is reliably
transformable by `torch_geometric.nn.to_hetero` (GCN assumes a single relation
type, GIN's inner MLP collapses on bipartite graphs).
